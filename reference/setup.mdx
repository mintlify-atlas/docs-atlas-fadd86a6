## Overview

DigiForum is built with modern web technologies optimized for fast development and accessibility. This guide will help you set up your local development environment.

## Tech Stack

DigiForum uses the following technologies:

<CardGroup cols={2}>
  <Card title="React 18" icon="react" iconType="duotone">
    Modern UI library with hooks and concurrent features
  </Card>
  <Card title="Vite" icon="bolt" iconType="duotone">
    Lightning-fast build tool with HMR
  </Card>
  <Card title="TypeScript" icon="code" iconType="duotone">
    Type-safe JavaScript for better code reliability
  </Card>
  <Card title="Tailwind CSS" icon="palette" iconType="duotone">
    Utility-first CSS framework for responsive design
  </Card>
</CardGroup>

### Core Libraries

- **shadcn/ui** - Accessible component library built on Radix UI
- **React Router** - Client-side routing
- **React Hook Form** - Form validation with Zod
- **TanStack Query** - Data fetching and caching
- **Lucide React** - Icon library
- **next-themes** - Theme management
- **Recharts** - Data visualization

## Installation

<Steps>
  <Step title="Clone the repository">
    Clone the DigiForum repository to your local machine:

    ```bash
    git clone <YOUR_GIT_URL>
    cd vite_react_shadcn_ts
    ```
  </Step>

  <Step title="Install dependencies">
    Install the project dependencies using your preferred package manager:

    <CodeGroup>
      ```bash npm
      npm install
      ```

      ```bash yarn
      yarn install
      ```

      ```bash pnpm
      pnpm install
      ```
    </CodeGroup>
  </Step>

  <Step title="Start the development server">
    Run the development server with hot module replacement:

    <CodeGroup>
      ```bash npm
      npm run dev
      ```

      ```bash yarn
      yarn dev
      ```

      ```bash pnpm
      pnpm dev
      ```
    </CodeGroup>

    The application will be available at `http://localhost:8080`
  </Step>
</Steps>

## Project Structure

The project uses Vite's standard structure with path aliasing:

```
src/
├── components/     # Reusable UI components
├── pages/          # Page components
├── hooks/          # Custom React hooks
├── lib/            # Utility functions
├── styles/         # Global styles
└── main.tsx        # Application entry point
```

### Path Aliases

The project uses `@/` as an alias for the `src/` directory:

```typescript
import { Button } from '@/components/ui/button'
import { cn } from '@/lib/utils'
```

## Development Configuration

### Vite Configuration

The development server is configured with:

- **Host**: `::` (listens on all network interfaces)
- **Port**: `8080`
- **Plugins**: 
  - `@vitejs/plugin-react-swc` - Fast Refresh using SWC
  - `lovable-tagger` - Component tagging (development only)

### TypeScript

The project uses TypeScript for type safety. Type definitions are available for all dependencies.

### Tailwind CSS

Tailwind is configured with:
- Custom color scheme for accessibility
- Typography plugin for rich text content
- Animation utilities via `tailwindcss-animate`

## Data Persistence

<Warning>
  DigiForum currently uses **LocalStorage** for client-side data persistence. All user data is stored locally in the browser.
</Warning>

This means:
- No backend server is required for development
- User data persists across sessions on the same device
- Data is not synchronized across devices

## Troubleshooting

<AccordionGroup>
  <Accordion title="Port 8080 is already in use">
    If port 8080 is already in use, you can:
    
    1. Stop the process using port 8080
    2. Modify the port in `vite.config.ts`:
    
    ```typescript
    server: {
      host: "::",
      port: 3000, // Change to your preferred port
    }
    ```
  </Accordion>

  <Accordion title="Module not found errors">
    Try deleting `node_modules` and reinstalling:
    
    ```bash
    rm -rf node_modules
    npm install
    ```
  </Accordion>

  <Accordion title="TypeScript errors">
    Ensure you're using TypeScript 5.8.3 or higher:
    
    ```bash
    npm list typescript
    ```
    
    If needed, reinstall dependencies:
    
    ```bash
    npm install
    ```
  </Accordion>
</AccordionGroup>
