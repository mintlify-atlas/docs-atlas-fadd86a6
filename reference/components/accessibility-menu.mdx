## Overview

The `AccessibilityMenu` component provides a comprehensive set of accessibility features specifically designed for users with physical disabilities. It offers a floating button with a dropdown menu containing various accessibility settings that persist across sessions.

## Settings Interface

```typescript
interface AccessibilitySettings {
  fontSize: number;        // 80-150, default: 100
  highContrast: boolean;   // default: false
  readAloud: boolean;      // default: false
  accessibleFont: boolean; // default: false
  largeCursor: boolean;    // default: false
}
```

## Feature Details

### Font Size Adjustment

<Card title="Font Size Control" icon="text-size">
  **Range:** 80% - 150%
  
  **Increment:** 10%
  
  **Implementation:** Modifies `document.documentElement.style.fontSize`
  
  **Feedback:** Toast notification showing current percentage
</Card>

**Functions:**
- `increaseFontSize()` - Increases font size by 10% (max 150%)
- `decreaseFontSize()` - Decreases font size by 10% (min 80%)

### High Contrast Mode

<Card title="High Contrast Toggle" icon="circle-half-stroke">
  **CSS Class:** `.high-contrast`
  
  **Scope:** Applied to `document.body`
  
  **Feedback:** Toast showing "Alto contraste activado" or "Modo normal activado"
</Card>

This feature adds a `high-contrast` class to the body element, which should be styled in your CSS to provide better contrast ratios.

### Read Aloud Feature

<Card title="Text-to-Speech" icon="volume-high">
  **Technology:** Web Speech API (`SpeechSynthesisUtterance`)
  
  **Language:** Spanish (es-ES)
  
  **Speech Rate:** 0.9 (slightly slower for clarity)
  
  **Trigger:** Text selection via mouseup event
</Card>

**Behavior:**
- When enabled, automatically reads selected text aloud
- Shows "(leyendo...)" status indicator while speaking
- Cancels previous speech when new text is selected
- Automatically cleans up on component unmount

### Accessible Font

<Card title="Font Toggle" icon="font">
  **CSS Class:** `.accessible-font`
  
  **Scope:** Applied to `document.body`
  
  **Purpose:** Switches to a more readable font (e.g., OpenDyslexic, Comic Sans)
</Card>

### Large Cursor

<Card title="Cursor Size" icon="mouse-pointer">
  **CSS Class:** `.large-cursor`
  
  **Scope:** Applied to `document.body`
  
  **Implementation:** CSS should define larger cursor styles
</Card>

## Persistence

All settings are automatically saved to `localStorage` under the key `accessibility-settings`:

```typescript
// Settings are saved on every update
localStorage.setItem("accessibility-settings", JSON.stringify(newSettings));

// Settings are loaded on component mount
const saved = localStorage.getItem("accessibility-settings");
if (saved) {
  const parsed = JSON.parse(saved);
  setSettings(parsed);
  applySettings(parsed);
}
```

### CSS Requirements

You must define CSS styles for the following classes in your global stylesheet:

```css
/* High Contrast Mode */
body.high-contrast {
  --background: #000;
  --foreground: #fff;
  /* Adjust other color variables for high contrast */
}

/* Accessible Font */
body.accessible-font {
  font-family: 'OpenDyslexic', 'Comic Sans MS', sans-serif;
}

/* Large Cursor */
body.large-cursor,
body.large-cursor * {
  cursor: url('large-cursor.cur'), auto !important;
  /* Or use CSS to scale cursor */
}
```

## Event Handling

The component uses several event listeners:

1. **mouseup** - Detects text selection for Read Aloud feature
2. **Component mount** - Loads saved settings from localStorage
3. **Component unmount** - Cleans up event listeners and stops speech synthesis

## State Management

```typescript
const [settings, setSettings] = useState<AccessibilitySettings>(DEFAULT_SETTINGS);
const [isReading, setIsReading] = useState(false);
```

- `settings` - Current accessibility configuration
- `isReading` - Indicates if text-to-speech is currently active
