## Overview

## Commonly Used Components

### Button

The Button component is the most widely used component in DigiForum, featuring multiple variants and sizes.

**Location:** `~/workspace/source/src/components/ui/button.tsx`

#### Props Interface

```typescript
export interface ButtonProps
  extends React.ButtonHTMLAttributes<HTMLButtonElement>,
    VariantProps<typeof buttonVariants> {
  asChild?: boolean;
}
```

#### Usage Examples

```typescript
import { Button } from "@/components/ui/button";

// Default button
<Button>Click me</Button>

// Outline variant, small size
<Button variant="outline" size="sm">Cancel</Button>

// Ghost button with icon
<Button variant="ghost" size="icon">
  <Settings className="h-4 w-4" />
</Button>

// As child (renders as custom component)
<Button asChild>
  <Link to="/dashboard">Go to Dashboard</Link>
</Button>
```

## Best Practices

<Tip>
  Always use the `asChild` prop when wrapping components like Links inside Buttons to avoid invalid HTML nesting.
</Tip>

<Check>
  Leverage the built-in variants and sizes before creating custom styles to maintain consistency.
</Check>

<Check>
  All form inputs should be paired with Label components using proper `htmlFor` and `id` attributes for accessibility.
</Check>

<Warning>
  Don't modify the base component files in `components/ui/` directly. Instead, create wrapper components or use className props for customization.
</Warning>
