## Overview

DigiForum provides rich social interaction features that enable community members to engage with posts through likes, comments, and bookmarking. These features foster community engagement and help users track content that resonates with them.

## Liking Posts

### Toggle Like Functionality

Likes work as a toggle - clicking again removes the like:

```typescript storage.ts:246-271
export const togglePostLike = (postId: string, userId: string): { liked: boolean; totalLikes: number } => {
  const likes = getPostLikes();
  const existingLikeIndex = likes.findIndex(
    like => like.postId === postId && like.userId === userId
  );

  if (existingLikeIndex !== -1) {
    // Quitar like
    likes.splice(existingLikeIndex, 1);
    localStorage.setItem("postLikes", JSON.stringify(likes));
    const totalLikes = likes.filter(like => like.postId === postId).length;
    return { liked: false, totalLikes };
  } else {
    // Agregar like
    const newLike: PostLike = {
      id: `like_${Date.now()}_${Math.random()}`,
      postId,
      userId,
      createdAt: new Date().toISOString(),
    };
    likes.push(newLike);
    localStorage.setItem("postLikes", JSON.stringify(likes));
    const totalLikes = likes.filter(like => like.postId === postId).length;
    return { liked: true, totalLikes };
  }
};
```

## Commenting on Posts

### Adding Comments

Users can add comments to posts from the PostDetail page:

```typescript PostDetail.tsx:54-80
const handleAddComment = (e: React.FormEvent) => {
  e.preventDefault();
  
  if (!newComment.trim()) {
    toast.error("Por favor escribe un comentario");
    return;
  }

  if (!currentUser || !id) {
    toast.error("Error al agregar comentario");
    return;
  }

  const comment: Comment = {
    id: Date.now().toString(),
    postId: id,
    authorId: currentUser.id,
    authorName: currentUser.fullName,
    content: newComment.trim(),
    createdAt: new Date().toISOString(),
  };

  addComment(comment);
  toast.success("Comentario agregado");
  setNewComment("");
  loadComments(id);
};
```

## Saving Posts

### Toggle Save Functionality

Similar to likes, saves work as a toggle:

```typescript storage.ts:290-313
export const toggleSavedPost = (postId: string, userId: string): { saved: boolean } => {
  const savedPosts = getSavedPosts();
  const existingSaveIndex = savedPosts.findIndex(
    save => save.postId === postId && save.userId === userId
  );

  if (existingSaveIndex !== -1) {
    // Quitar guardado
    savedPosts.splice(existingSaveIndex, 1);
    localStorage.setItem("savedPosts", JSON.stringify(savedPosts));
    return { saved: false };
  } else {
    // Guardar publicación
    const newSave: SavedPost = {
      id: `save_${Date.now()}_${Math.random()}`,
      postId,
      userId,
      createdAt: new Date().toISOString(),
    };
    savedPosts.push(newSave);
    localStorage.setItem("savedPosts", JSON.stringify(savedPosts));
    return { saved: true };
  }
};
```

```typescript getUserSavedPosts()
export const getUserSavedPosts = (userId: string): Post[] => {
  const savedPosts = getSavedPosts();
  const posts = getPosts();
  
  const userSavedPostIds = savedPosts
    .filter(save => save.userId === userId)
    .map(save => save.postId);
  
  return posts.filter(post => userSavedPostIds.includes(post.id));
};
```

## Best Practices

<AccordionGroup>
  <Accordion title="Like Management">
    - Likes are user-specific and persist across sessions
    - The toggle mechanism prevents duplicate likes
    - Total like counts are calculated dynamically
    - UI updates immediately after like/unlike actions
  </Accordion>
  
  <Accordion title="Comment Management">
    - Comments are immutable once posted (no edit/delete in current version)
    - Comments preserve whitespace and line breaks
    - Empty comments are rejected with validation
    - Timestamps show full date and time for context
  </Accordion>
  
  <Accordion title="Save Management">
    - Saved posts are private to each user
    - The toggle mechanism prevents duplicate saves
    - Saved posts can be viewed from the profile page
    - Saves persist even if the original post is modified
  </Accordion>
  
  <Accordion title="UI/UX Considerations">
    - All interactions provide immediate visual feedback
    - Toast notifications confirm successful actions
    - Icons and emojis provide intuitive visual indicators
    - Counts update in real-time after interactions
  </Accordion>
</AccordionGroup>
