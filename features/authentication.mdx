## Overview

DigiForum implements a complete authentication system that allows users to register, login, manage their sessions, and maintain profile data. The authentication system is built using localStorage for data persistence and React hooks for state management.

## User Data Model

The User interface defines the structure of user data in the application:

```typescript storage.ts:2-12

  username: string;

  dateOfBirth?: string;

  registeredAt: string;

```

## Registration Flow

### Registration Form

The registration form in `Auth.tsx` collects user information with comprehensive validation:

```typescript Auth.tsx:75-143
const handleRegister = (e: React.FormEvent) => {
  e.preventDefault();
  setError("");
  setSuccess("");

  // Validations
  if (!fullName || !email || !username || !password || !confirmPassword) {
    setError("Por favor completa todos los campos obligatorios");
    return;
  }

  if (!/^[^\s@]+@[^\s@]+\.[^\s@]+$/.test(email)) {
    setError("Por favor ingresa un correo electrónico válido");
    return;
  }

  if (password.length < 8) {
    setError("La contraseña debe tener al menos 8 caracteres");
    return;
  }

  if (password !== confirmPassword) {
    setError("Las contraseñas no coinciden");
    return;
  }

  const users = getUsers();
  
  if (users.some(u => u.email === email)) {
    setError("Ya existe una cuenta con este correo electrónico");
    return;
  }

  if (users.some(u => u.username === username)) {
    setError("Este nombre de usuario ya está en uso");
    return;
  }

  // Crear nuevo usuario
  const newUser: User = {
    id: Date.now().toString(),
    email,
    username,
    fullName,
    password,
    amputationType: amputationType || undefined,
    dateOfBirth: dateOfBirth ? dateOfBirth.toISOString() : undefined,
    registeredAt: new Date().toISOString(),
  };

  addUser(newUser);
  setSuccess("¡Cuenta creada exitosamente! Ahora puedes iniciar sesión.");
  toast.success("Cuenta creada exitosamente");
  
  // Limpiar formulario y cambiar a login
  // ...
  setTimeout(() => {
    setMode("login");
    setSuccess("");
  }, 2000);
};
```

## Login Flow

### Flexible Login

Users can login with either their email or username:

```typescript Auth.tsx:52-73
const handleLogin = (e: React.FormEvent) => {
  e.preventDefault();
  setError("");
  
  if (!loginEmail || !loginPassword) {
    setError("Por favor completa todos los campos");
    return;
  }

  const users = getUsers();
  const user = users.find(
    u => (u.email === loginEmail || u.username === loginEmail) && u.password === loginPassword
  );

  if (user) {
    setCurrentUser(user.id);
    toast.success(`¡Bienvenido/a de nuevo, ${user.fullName}!`);
    navigate("/dashboard");
  } else {
    setError("Credenciales incorrectas. Por favor verifica tu correo/usuario y contraseña.");
  }
};
```

## Session Management

### Storage Functions

The session is managed through localStorage with these key functions:

```typescript storage.ts:185-198
export const getCurrentUser = (): User | null => {
  const currentUserId = localStorage.getItem("currentUser");
  if (!currentUserId) return null;
  const users = getUsers();
  return users.find(u => u.id === currentUserId) || null;
};

export const setCurrentUser = (userId: string) => {
  localStorage.setItem("currentUser", userId);
};

export const logout = () => {
  localStorage.removeItem("currentUser");
};
```

### Protected Routes

The `ProtectedRoute` component ensures users are authenticated:

```typescript ProtectedRoute.tsx
import { Navigate } from "react-router-dom";
import { getCurrentUser } from "@/lib/storage";

const ProtectedRoute = ({ children }: { children: React.ReactNode }) => {
  const currentUser = getCurrentUser();
  
  if (!currentUser) {
    return <Navigate to="/auth?mode=login" replace />;
  }
  
  return <>{children}</>;
};
```

### Session Checks

Users can update their profile with validation:

```typescript Profile.tsx:81-136
const handleSaveProfile = () => {
  if (!editName.trim()) {
    toast.error("El nombre no puede estar vacío");
    return;
  }

  // Validar contraseña si se está intentando cambiar
  if (currentPassword || newPassword || confirmNewPassword) {
    if (!currentPassword) {
      toast.error("Debes ingresar tu contraseña actual");
      return;
    }
    if (newPassword.length < 6) {
      toast.error("La nueva contraseña debe tener al menos 6 caracteres");
      return;
    }
    if (newPassword !== confirmNewPassword) {
      toast.error("Las contraseñas no coinciden");
      return;
    }
  }

  const users = getUsers();
  const userIndex = users.findIndex(u => u.id === currentUser.id);
  
  if (userIndex !== -1) {
    users[userIndex].fullName = editName.trim();
    users[userIndex].amputationType = editAmputationType || undefined;
    users[userIndex].dateOfBirth = editDateOfBirth?.toISOString();
    users[userIndex].profilePhoto = editProfilePhoto.trim() || undefined;
    
    if (newPassword) {
      users[userIndex].password = newPassword;
    }
    
    localStorage.setItem("users", JSON.stringify(users));
    setCurrentUser(users[userIndex]);
    toast.success("Perfil actualizado exitosamente");
  }
};
```

const handlePhotoChange = (e: React.ChangeEvent<HTMLInputElement>) => {
  const file = e.target.files?.[0];

## Security Considerations

<Warning>
  The current authentication implementation stores passwords in plain text in localStorage. This is suitable for a prototype/demo but **should not be used in production**. For production applications:
  - Use proper backend authentication with JWT tokens
  - Hash passwords using bcrypt or similar
  - Implement HTTPS
  - Add session expiration
  - Consider OAuth providers
</Warning>
