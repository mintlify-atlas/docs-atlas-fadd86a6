## Overview

Posts are the core content type in DigiForum, allowing community members to share personal experiences, ask questions, share resources, and provide emotional support. The platform includes comprehensive post creation, categorization, filtering, and search functionality.

## Post Data Model

The Post interface defines the structure of forum posts:

```typescript storage.ts:14-22
export interface Post {
  id: string;
  title: string;
  category: string;
  content: string;
  authorId: string;
  authorName: string;
  createdAt: string;
}
```

### Field Descriptions

<ParamField path="id" type="string" required>
  Unique identifier for the post (timestamp-based)
</ParamField>

<ParamField path="title" type="string" required>
  Post title - should be descriptive and concise
</ParamField>

<ParamField path="category" type="string" required>
  Category classification for the post (see Categories section)
</ParamField>

<ParamField path="content" type="string" required>
  Full post content - supports multi-line text with whitespace preservation
</ParamField>

<ParamField path="authorId" type="string" required>
  User ID of the post author
</ParamField>

<ParamField path="authorName" type="string" required>
  Display name of the post author
</ParamField>

<ParamField path="createdAt" type="string" required>
  ISO timestamp of when the post was created
</ParamField>

## Creating Posts

### Post Creation Form

The post creation interface is embedded in the Dashboard with a collapsible form:

```typescript Dashboard.tsx:76-110
const handleCreatePost = (e: React.FormEvent) => {
  e.preventDefault();
  
  if (!title.trim() || !category || !content.trim()) {
    toast.error("Por favor completa todos los campos");
    return;
  }

  if (!currentUser) {
    toast.error("Debes iniciar sesión para crear publicaciones");
    return;
  }

  const newPost: Post = {
    id: Date.now().toString(),
    title: title.trim(),
    category,
    content: content.trim(),
    authorId: currentUser.id,
    authorName: currentUser.fullName,
    createdAt: new Date().toISOString(),
  };

  addPost(newPost);
  toast.success("Publicación creada exitosamente");
  
  // Reset form
  setTitle("");
  setCategory("");
  setContent("");
  setIsCreating(false);
  
  // Reload posts
  loadPosts();
};
```

## Search and Filtering

### Search Functionality

Users can search posts by title, content, category, or author name:

```typescript Dashboard.tsx:47-67
useEffect(() => {
  let filtered = posts;

  // Filter by category
  if (selectedCategory !== "all") {
    filtered = filtered.filter(post => post.category === selectedCategory);
  }

  // Filter by search query
  if (searchQuery.trim() !== "") {
    const query = searchQuery.toLowerCase();
    filtered = filtered.filter(post => 
      post.title.toLowerCase().includes(query) ||
      post.content.toLowerCase().includes(query) ||
      post.category.toLowerCase().includes(query) ||
      post.authorName.toLowerCase().includes(query)
    );
  }

  setFilteredPosts(filtered);
}, [searchQuery, selectedCategory, posts]);
```

## Storage Functions

### Post Management API

<CodeGroup>

```typescript getPosts()
export const getPosts = (): Post[] => {
  const posts = localStorage.getItem("posts");
  return posts ? JSON.parse(posts) : [];
};
```

```typescript addPost()
export const addPost = (post: Post) => {
  const posts = getPosts();
  posts.unshift(post); // Add to beginning for reverse chronological order
  localStorage.setItem("posts", JSON.stringify(posts));
};
```

```typescript getPostById()
export const getPostById = (id: string): Post | null => {
  const posts = getPosts();
  return posts.find(p => p.id === id) || null;
};
```

```typescript getPostsByAuthor()
export const getPostsByAuthor = (authorId: string): Post[] => {
  const posts = getPosts();
  return posts.filter(p => p.authorId === authorId);
};
```

</CodeGroup>
